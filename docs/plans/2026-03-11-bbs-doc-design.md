# BBS 中文解释文档设计

日期：2026-03-11

## 目标

在 `docs` 中撰写一份大学生水平可理解的中文解释文档，覆盖：

- 按问题组织的导读
- 按章节的逐节解释
- 对关键概念、公式和结论的通俗说明
- 明确区分 `作者观点` 与 `理解与提醒`

## 采用方案

采用单一主文档方案，在一个文件中整合：

- 导读
- 逐节解释
- 术语表
- “证明了什么 / 主张了什么”的区分

原因：

- 用户明确希望两种阅读方式都保留
- 单文件更适合首次阅读
- 便于后续继续补充

## 写作原则

- 不做逐句翻译，改做教学型讲解
- 尽量将“定义-直觉-例子-边界”放在一起
- 对系统级主张保留适度提醒，但不喧宾夺主
- 文件命名使用 ASCII，便于后续维护

## 产出

- 主文档：`docs/bbs-paper-explained.md`
